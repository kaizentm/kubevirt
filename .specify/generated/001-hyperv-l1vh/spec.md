## Primary User Story

As a cluster operator running an L1VH-capable Azure cluster, I want to enable Hyper-V L1VH support cluster-wide so that all VMs automatically benefit from near-native performance and direct hardware access.

## Acceptance Scenarios

1. Given an L1VH-capable homogeneous cluster with feature gate enabled, When a standard VirtualMachine is created, Then it boots using L1VH backend transparently.
2. Given feature gate disabled, When a VirtualMachine is created, Then behavior is identical to pre-feature KubeVirt.
3. Given feature gate enabled on L1VH cluster, When VM lifecycle operations (start/stop/restart/delete) are invoked, Then they behave identically to KVM-based VMs.
4. Given feature gate enabled and all nodes L1VH-capable, When a VM migrates between nodes, Then live migration succeeds with no spec changes.
